## Hero 动画：Glitch 故障风转场 v3

把之前平淡的"卷起 + clip-path"转场替换为 **Glitch 故障风**，效果震撼。

### 实现要点
- **8 条 `.hero-glitch-bar`** 错峰向上飞出（横向偏移 `(i-3.5) * 14px`、纵向 `-95vh - i*9vh`、旋转 `±8deg`）
- **SVG filter `#hero-rgb-split`** 用 feColorMatrix 把红绿蓝三通道分离，红左移 4px、蓝右移 4px、绿原位
- **`.hero-noise-spray`** 全屏噪点纹理，0.4s steps(8) animation 模拟扫描
- **文字破碎**：hero-text-headline skewY(-10deg) + translateY(-32vh)，副标题反向 skew
- **CTA 反向缩放飞出**

### 关键 CSS 技巧
- `mix-blend-mode: screen` 让 glitch bar 和 canvas 颜色叠加发光（无需复制 canvas 视觉）
- `feBlend mode="screen"` 在 SVG filter 内合成三通道
- 用 `--trigger = max(0, calc(var(--p) - 0.12))` 让所有 Glitch bar 在 0.12 才开始动，避免 0% 时就出戏
- `hue-rotate(calc(var(--i) * 22deg))` 给每条 bar 不同的色调，更"故障"

### 关键文件
- `scripts/hero-inject.js`：完整转场 CSS + SVG filter + glitch layer HTML
- `source/hero/river-hero.js?v=5`：含 visibilitychange 处理（之前已修复）

- 25%：glitch bar 飞出，文字模糊偏移
- 35-45%：撕裂感最强，标题飞散
- 50%：博客内容浮现

## v7「弹簧蓄力→爆裂释放」重构（21:10 commit 3a4fe2b6）

用户第三次要求重构转场，明确"先有阻力再下滑"。
用户同时要求清理首页：浮动播放器、右下角按钮、blog-info、site-info、scroll-down 全删，只留 hero + 顶部 nav。

### 阻力→释放数学模型
- 临界点 raw = 0.45（45% 滚动量时 hero 蓄满势能）
- **蓄力段**（raw ∈ [0, 0.45]）：
  - 视觉进度用 `easeInQuad(pin) × 0.22`（前慢后快蓄能）
  - shell `translateY 0 → +7vh`（被往下"压"）
  - shell `scale 1.0 → 1.08`（鼓胀感）
  - 文字 `translateY 0 → -2vh`（轻微被弹簧拉起）
  - 进度导轨慢填充（脉动圆点存在）
- **释放段**（raw ∈ [0.45, 1.0]）：
  - 视觉进度用 `0.22 + easeOutExpo × 0.78`（前 15% 缓、后 85% 极速）
  - shell `translateY +7vh → -200vh`（飞出屏幕）
  - shell `scale 1.08 → 1.0`
  - 文字散开 ±25vh + 飞走 -60vh（headline 左 / cta 右 / quote 中）
  - main 从 +100vh 滑入 + 淡入
  - 浮点播放器等被 `display:none`

### CSS 关键设计
- `--hero-pin` / `--hero-easeout` 两个独立变量驱动两段
- shell `transform: translate3d(0, pin*7vh - ease*207vh, 0)`
- section.scale `1.0 + pin*0.08 - ease*0.08`
- 文字 alternate direction（headline 左、cta 右）增强爆裂感
- 博客 main z-index 1100 在 hero-shell 1000 之上，避免闪烁

### 性能铁律（v3→v4→v5→v6→v7 经验）
- ❌ 不在 hero-shell 上用 CSS filter / SVG filter / mix-blend-mode / will-change
- ✅ 只用 transform: translate3d + scale + opacity
- ✅ canvas 单独加 translateZ(0)
- ✅ 0 transition → 滚动 100% 跟手
