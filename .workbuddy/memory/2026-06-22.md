## Phase 9.1: 锐化修复（22:50）

**用户反馈**：视频里"画面模糊、有黑色硬边"

### 根因 + 修复
1. **底部黑色硬边 16px**
   - 根因：`.hero` CSS `height: 100vh; min-height: 640px`，但 viewport 只有 624px → canvas 1262×640（高于 viewport 16px）
   - 修复：CSS `min-height` 删掉 + river-hero.js `size()` 用 `window.innerWidth/Height` 而非 `hero.clientWidth/Height`
   - 效果：canvas 严格匹配 viewport，黑色硬边消失

2. **整体颗粒感过强、模糊**
   - 根因 1：river-hero.js `CELL_PX = 6`（6px 字符 cell）稀疏
   - 修复 1：`CELL_PX = 4`（4px 细密）
   - 根因 2：`dpr = Math.min(window.devicePixelRatio || 1, 1.0) * liveScale` 强制 cap 1.0
   - 修复 2：`dpr = Math.min(window.devicePixelRatio || 1, 1.5) * liveScale`（retina 友好）
   - 效果：颗粒感降到合理水平，跟 river.ai 原版接近

3. **FPS 跳动 12-60**
   - 根因：`canyonMaxSteps = 148` ray-march 步数 + `canyonStepScale = 1.0` + `frameCap = 40`
   - 修复：`canyonMaxSteps = 112` + `canyonStepScale = 1.18` + `frameCap = 50`
   - 效果：FPS 稳定 65-78

- `source/hero/index.html`：删 min-height:640px + 调试 slider 默认值
