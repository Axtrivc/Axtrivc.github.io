## River Hero Phase 7：立体光照 + 竖向字符流（22:04）

用户反馈：山脉有层次感但没立体感；太阳不够亮；字符河可以像 river.ai 那样竖向点缀。

### 研究了 river.ai 的真实算法
读了 `river-script.js` 1360-1420 行，发现他们的立体感来自：
1. **真实 terrain texture** + raymarch（过于复杂，不抄）
2. **关键 trick**：`fwidth` 算 normal + Lambert 漫反射 + hillshade + cast shadow + AO
3. **太阳方向**：基于 sunX/elev/time-of-day，不是固定的

### Phase 7 实现
- **Lambert 光照**（核心立体感来源）：
  - 对每个山层采样高度 + 左右偏移采样 → 求偏导 → 构造 normal
  - `dot(normal, lightDir)` → hillshade factor（受光面亮，背光面暗）
  - 山体颜色：`mix(C_MT_SHADE, C_MT_LIT, hill)`
- **太阳更亮**：
  - radius 0.040（稍大）
  - 圆盘亮度从 1.2 → **2.0**（直接 +C_SUN_CORE * 2.0）
  - sunGlow 0.6 → 0.7
  - 加 **God rays**（径向噪声条纹，从太阳辐射）
- **竖向字符流**（点缀）：
  - 60 列，每列独立速度
  - 字符从上往下"流"（mod 循环）
  - 头尾淡入淡出（像雨滴）
  - 只有 ~45% 的列有可见字符（随机稀疏）
  - 颜色用 C_CHAR 浅蓝灰
